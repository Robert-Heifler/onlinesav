How It Solves These Problems

Backorders Report: The Backorders report is driven by a query that joins sales orders, customers, order lines, and products, then filters by order date range, optional customer name, and an optional Backorders Only prompt that returns only lines where Backorder=True, showing the exact customer, product, quantities, and line totals to resolve shortages quickly.
InvValue Report: The InvValue report calculates RetailValue, CostValue, and GrossMarginValue from UnitPrice, Cost, and QuantityOnHand (with Nz safeguards) and lets users filter by SupplierName and Category, making it easy to see where money is tied up and which SKUs or categories are driving or hurting margin.
SalesPaySts Report: The SalesPaySts report groups sales by PaymentStatus and order within a selectable date range and sums line totals, so you can quickly see how much revenue sits in each status and focus follow-up and deposits where it matters.
